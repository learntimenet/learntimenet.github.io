💡 Qu'est-ce qu'une expression lambda ?

Une expression lambda est un bref bloc de code qui prend des paramètres et renvoie un résultat - un peu comme une méthode sans nom.

Introduit dans Java 8, il prend en charge le style de programmation fonctionnelle en Java.

📏 Syntaxe

(parameters) -> expression

Ou pour plusieurs déclarations :

(parameters) -> {
    // code
    return result;
}

🧪 Exemple simple

Runnable r = () -> System.out.println("Running a thread");
r.run();

✅ Remplace le modèle de classes internes anonymes ✅ Améliore la lisibilité et la concision

🔁 Avec des interfaces fonctionnelles

Une expression lambda peut être utilisée uniquement lorsqu'il existe une interface fonctionnelle - une interface avec une seule méthode abstraite.

@FunctionalInterface
interface Greet {
    void sayHello();
}

public class Demo {
    public static void main(String[] args) {
        Greet g = () -> System.out.println("Hello!");
        g.sayHello();
    }
}

🧠 Interfaces fonctionnelles communes (de java.util.function)

. . .
Interface Méthode Description
Consommateur<T> accept(T t) Prend en compte les données, ne renvoie rien
Fournisseur<T> get() Pas d'entrée, renvoie quelque chose
Fonction<T,R> apply(T t) Prend des données en entrée, renvoie des données en sortie
Prédicat<T> test(T t) Prend en compte les données, renvoie un booléen

🧹 Lambdas avec Collections

forEach()

List<String> names = List.of("Alice", "Bob", "Charlie");
names.forEach(name -> System.out.println(name));

filter(), map(), collect()

List<String> names = List.of("Anna", "Bella", "Chris");

List<String> result = names.stream()
    .filter(name -> name.startsWith("A"))
    .map(String::toUpperCase)
    .collect(Collectors.toList());

System.out.println(result); // [ANNA]

Streams + lambdas = magie ✨

🔄 Méthode Références

Une manière plus courte d'écrire des lambdas lorsque vous appelez des méthodes existantes :

names.forEach(System.out::println);

📘 Recap

  • Les lambdas rendent le code Java plus expressif et plus concis
  • Travailler avec des interfaces fonctionnelles (comme Runnable, Predicate)
  • Combinaison avec des collections et des flux pour un traitement propre des données
  • Utiliser des références de méthode pour plus de concision
