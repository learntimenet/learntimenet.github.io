⚙️ Qu'est-ce que le multithreading ?

Le multithreading consiste à exécuter plusieurs fils d'exécution simultanément - comme le multitâche pour votre programme.

✅ Améliore les performances ✅ Gère les tâches lourdes sans geler l'interface utilisateur ✅ Essentiel pour les applications réactives en temps réel

👯‍♂️ Création d'un fil de discussion - Deux façons de procéder

1. Extension de la classe **Thread

class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start();  // starts new thread
    }
}

2. Mise en œuvre de Runnable

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable running");
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t2 = new Thread(new MyRunnable());
        t2.start();
    }
}

✅ Runnable est préféré dans le Java moderne pour la flexibilité et un code plus propre.

💤 Thread Sleep

Interrompre temporairement un fil de discussion :

Thread.sleep(1000);  // 1 second (in milliseconds)

⏱️ Joindre des fils de discussion

Attendre la fin d'un fil avant de continuer :

t1.join();  // waits for t1 to finish

🔄 Exécution de threads en parallèle

class Task implements Runnable {
    private String name;

    public Task(String name) {
        this.name = name;
    }

    public void run() {
        for (int i = 1; i <= 3; i++) {
            System.out.println(name + " - Step " + i);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t1 = new Thread(new Task("Task A"));
        Thread t2 = new Thread(new Task("Task B"));

        t1.start();
        t2.start();
    }
}

🛡️ Synchronisation - Éviter les conflits

Les threads peuvent provoquer une corruption de données s'ils accèdent en même temps à des ressources partagées.

Utiliser synchronized :

public synchronized void increment() {
    count++;
}

🧠 Thread Pools avec ExecutorService

import java.util.concurrent.*;

ExecutorService executor = Executors.newFixedThreadPool(2);

executor.submit(() -> System.out.println("Task running"));

executor.shutdown();

✅ Efficace pour gérer de nombreuses petites tâches.

📘 Recap

  • Les threads exécutent le code en parallèle, ce qui améliore les performances.
  • Utiliser Thread ou Runnable pour définir les tâches
  • synchronized évite les problèmes de données
  • ExecutorService est idéal pour gérer de nombreuses tâches
