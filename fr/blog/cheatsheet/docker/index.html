Introduction

Docker permet d'empaqueter et d'exécuter une application dans un environnement faiblement isolé appelé conteneur. L'isolement et la sécurité permettent d'exécuter simultanément de nombreux conteneurs sur un hôte donné. Les conteneurs sont légers et contiennent tout ce qui est nécessaire à l'exécution de l'application, de sorte que vous n'avez pas besoin de vous fier à ce qui est actuellement installé sur l'hôte. Vous pouvez facilement partager des conteneurs pendant que vous travaillez, et être sûr que toutes les personnes avec lesquelles vous partagez obtiennent le même conteneur qui fonctionne de la même manière.

Commandes générales

cheatsheet

  • Démarrer le démon docker
    docker -d
  • Obtenir de l'aide sur Docker. Vous pouvez également utiliser -help pour toutes les sous-commandes
    docker --help
  • Affichage des informations sur le système
    docker info

Images

Les images Docker sont un ensemble de logiciels légers, autonomes et exécutables qui comprennent tout ce qui est nécessaire à l'exécution d'une application : le code, le moteur d'exécution, les outils système, les bibliothèques système et les paramètres.

cheatsheet

  • Construire une image à partir d'un fichier Docker
    docker build -t .
  • Construire une image à partir d'un fichier Docker sans le cache
    docker build -t . -no-cache
  • Liste des images locales
    images docker
  • Supprimer une image
    docker rmi
  • Supprimez toutes les images inutilisées
    docker image prune

Conteneurs

Un conteneur est une instance d'exécution d'une image Docker. Un conteneur fonctionnera toujours de la même manière, quelle que soit l'infrastructure. Les conteneurs isolent le logiciel de son environnement et garantissent qu'il fonctionne uniformément malgré les différences, par exemple entre le développement et la mise en scène.

cheatsheet

  • Créer et exécuter un conteneur à partir d'une image, avec un nom personnalisé:
    docker run --name
  • Exécuter un conteneur avec et publier le(s) port(s) d'un conteneur sur l'hôte. docker run -p :
  • Exécuter un conteneur en arrière-plan
    docker run -d
  • Démarrage ou arrêt d'un conteneur existant :
    docker start|stop (ou )
  • Supprimer un conteneur arrêté :
    docker rm
  • Ouvrir un shell à l'intérieur d'un conteneur en cours d'exécution :
    docker exec -it sh
  • Récupérer et suivre les logs d'un conteneur :
    docker logs -f
  • Pour inspecter un conteneur en cours d'exécution :
    docker inspect (ou )
  • Pour dresser la liste des conteneurs en cours d'exécution :
    docker ps
  • Liste de tous les conteneurs Docker (en cours d'exécution et arrêtés):
    docker ps --all
  • Afficher les statistiques d'utilisation des ressources
    stats des conteneurs docker

Docker HUB

Docker Hub est un service fourni par Docker pour trouver et partager des images de conteneurs avec votre équipe. Pour en savoir plus et trouver des images, rendez-vous sur https://hub.docker.com

cheatsheet

  • Se connecter à Docker
    docker login -u
  • Publier une image sur Docker Hub
    docker push /
  • Hub de recherche d'une image
    docker search
  • Tirer une image d'un hub Docker
    docker pull
