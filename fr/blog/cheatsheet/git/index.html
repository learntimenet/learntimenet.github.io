Introduction

Git est le système de contrôle de version le plus répandu. Il permet aux développeurs de suivre les modifications, de collaborer efficacement et de gérer les bases de code de manière performante. Que vous soyez un débutant ou un développeur expérimenté, une référence rapide aux commandes Git peut vous faire gagner du temps et rationaliser votre flux de travail. Cet aide-mémoire Git fournit une liste complète des commandes Git essentielles, des astuces et des meilleures pratiques.

Git Basics

Cette section couvre les commandes fondamentales dont vous avez besoin pour démarrer avec Git. Ces commandes vous permettent d'initialiser un dépôt, de suivre les modifications et de valider votre travail. Que vous mettiez en place un nouveau projet ou que vous cloniez un projet existant, ces commandes sont essentielles pour tout développeur.

cheatsheet

  • Créez un nouveau dépôt Git dans le répertoire de votre projet
    . git init
  • Cloner un dépôt existant à partir d'une URL distante
    . git clone
  • Afficher l'état de votre répertoire de travail
    git status
  • Mettre en place les changements pour le prochain commit
    . git add
  • Mettre en scène tous les changements
    git add .
  • Commettre les modifications par étapes avec un message
    . git commit -m "Votre message de livraison"
  • Afficher l'historique des livraisons
    git log

Branchement et fusion

Les branches et les fusions sont des fonctionnalités essentielles de Git qui permettent aux développeurs de travailler simultanément sur plusieurs fonctionnalités ou corrections sans affecter la base de code principale. Les branches sont légères et faciles à créer, ce qui les rend idéales pour isoler les modifications. La fusion combine les modifications de différentes branches, ce qui garantit une intégration harmonieuse du code.

cheatsheet

  • Créer une nouvelle branche
    git branch
  • Passer à une branche existante
    git checkout
  • Créer et basculer vers une nouvelle branche en une seule commande
    . git checkout -b
  • Voir toutes les branches dans le référentiel
    . branche git
  • Fusionner une branche dans la branche courante
    . git merge
  • Supprimer une branche
    git branch -d

Dépôts à distance

Les dépôts distants vous permettent de collaborer avec d'autres personnes en poussant et en tirant des modifications vers et depuis un dépôt partagé. Git facilite la gestion des dépôts distants, ce qui permet une collaboration transparente entre les équipes et les projets.

cheatsheet

  • Ajouter un dépôt à distance
    git remote add
  • Liste de tous les dépôts distants
    git remote -v
  • Récupérer les modifications d'un référentiel distant
    git fetch
  • Tirer les modifications d'une branche distante
    git pull
  • Transférer les modifications vers une branche distante
    git push
  • Supprimer un dépôt à distance
    git remote remove

Annuler les modifications

Les erreurs arrivent, et Git fournit plusieurs moyens d'annuler les changements. Que vous ayez besoin de dépiler un fichier, de revenir sur un commit, ou de réinitialiser votre dépôt à un état antérieur, Git vous couvre.

cheatsheet

  • Déstocker un fichier tout en gardant les modifications
    git reset
  • Revenir sur les modifications d'un fichier par rapport au dernier commit
    . git checkout --
  • Modifier le dernier commit
    git commit --amend
  • Revenir sur un commit spécifique
    git revert
  • Réinitialiser le référentiel à un commit précédent
    . git reset --hard

Mise en réserve

La mise en réserve permet de sauvegarder temporairement les modifications qui ne sont pas prêtes à être validées. C'est utile lorsque vous avez besoin de changer de branche ou de travailler sur quelque chose d'autre sans perdre votre progression actuelle.

cheatsheet

  • Stocker les modifications dans le répertoire de travail
    . git stash
  • Voir toutes les réserves
    git stash list
  • Appliquer la réserve la plus récente
    . git stash apply
  • Appliquer une réserve spécifique
    git stash apply stash@{}
  • Supprimez la réserve la plus récente
    . git stash drop
  • Supprimer une réserve spécifique
    git stash drop stash@{}

Journaux et historique

Les journaux Git fournissent un historique détaillé des livraisons, ce qui facilite le suivi des modifications, l'identification des problèmes et la compréhension de l'évolution de votre base de code. Vous pouvez afficher les journaux dans différents formats pour répondre à vos besoins.

cheatsheet

  • Afficher l'historique des livraisons
    git log
  • Affichage d'un historique simplifié des livraisons
    git log --oneline
  • Voir les modifications apportées dans un commit spécifique
    git show
  • Voir les changements entre deux commits
    . git diff

Étiquetage

Les balises sont utilisées pour marquer des points spécifiques dans l'historique de votre référentiel, tels que les versions ou les étapes. Ils permettent de référencer facilement les livraisons importantes.

cheatsheet

  • Créer une balise légère
    git tag
  • Créer un tag annoté
    git tag -a <nom_de_l'étiquette> -m "Message de l'étiquette"
  • Voir tous les tags
    git tag
  • Pousser des balises vers un dépôt distant
    git push --tags
  • Supprimer une balise
    git tag -d

Commandes Git avancées

Les commandes Git avancées fournissent des outils puissants pour gérer des flux de travail complexes, déboguer et optimiser votre référentiel. Ces commandes sont essentielles pour les développeurs expérimentés qui travaillent sur de grands projets.

cheatsheet

  • Rebaser la branche actuelle sur une autre branche
    . git rebase
  • Démarrer un rebasement interactif
    git rebase -i
  • Appliquer un commit spécifique à la branche en cours
    . git cherry-pick
  • Pousser des balises vers un dépôt distant
    git push --tags
  • Supprimer une balise
    git tag -d

Conseils et bonnes pratiques

Le respect des meilleures pratiques garantit un flux de travail Git fluide et efficace. Ces conseils vous aideront à éviter les pièges les plus courants et à tirer le meilleur parti des fonctionnalités de Git.

  • Rédiger des messages de validation significatifs** : Utilisez des messages de validation clairs et concis pour décrire les changements.
  • Utiliser les branches pour les fonctionnalités : Créez des branches distinctes pour les nouvelles fonctionnalités ou les corrections de bogues.
  • Pull Before Pushing : Tirez toujours les derniers changements avant de pousser pour éviter les conflits.
  • Vérifier les modifications avant de valider : Utilisez git diff pour revoir les modifications avant de les valider.
  • Utiliser .gitignore : Exclure les fichiers inutiles du contrôle de version en utilisant un fichier .gitignore.
