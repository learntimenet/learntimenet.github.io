📊 Qu'est-ce que ORDER BY ?

La clause ORDER BY est utilisée pour trier l'ensemble des résultats d'une requête par une ou plusieurs colonnes. Vous pouvez trier les résultats dans l'ordre ascendant (ASC) ou descendant (DESC).

🧱 Syntaxe

SELECT column1, column2
FROM table_name
ORDER BY column1 [ASC|DESC];
  • ASC (par défaut) - ordre croissant
  • DESC - ordre décroissant

🔼 Exemple : Ascendant vs Descendant

Trier les utilisateurs par nom (A-Z)

SELECT name, email
FROM users
ORDER BY name ASC;

Trier les utilisateurs par âge (le plus âgé en premier)

SELECT name, age
FROM users
ORDER BY age DESC;

📚 Tri sur plusieurs colonnes

Vous pouvez trier par plus d'une colonne. SQL effectuera un tri sur la première colonne, et pour les lignes ayant la même valeur, un tri sur la deuxième, et ainsi de suite.

SELECT name, department, salary
FROM employees
ORDER BY department ASC, salary DESC;

💡 Il s'agit d'un tri alphabétique par département, puis, au sein de chaque département, par salaire le plus élevé.

⚠️ Gestion des NULL

Les différentes bases de données SQL gèrent différemment les valeurs NULL lors du tri :

  • Certains placent les NULL en premier dans l'ordre croissant
  • D'autres les placent en dernier

Il est souvent possible de contrôler ce comportement :

-- Example in PostgreSQL
ORDER BY column_name ASC NULLS LAST;

🚦 Conseil de performance

Le tri de grands ensembles de données peut ralentir vos requêtes. Pour de meilleures performances :

  • ✅ Utilisez des index sur les colonnes que vous ordonnez par
  • ✅ Éviter les tris inutiles si l'ordre n'a pas d'importance
  • 🚫 Ne pas trier par expressions sauf si nécessaire

📘 Recap

  • ORDER BY vous permet de trier les résultats de votre requête
  • Utiliser ASC (par défaut) ou DESC pour la direction.
  • Vous pouvez trier sur plusieurs colonnes pour un contrôle précis
  • Soyez attentif à la façon dont votre base de données gère les NULL.
  • Une commande efficace améliore l'expérience de l'utilisateur dans les rapports et les tableaux de bord.
