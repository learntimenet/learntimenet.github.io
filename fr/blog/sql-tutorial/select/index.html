🎯 Qu'est-ce que l'instruction SELECT ?

L'instruction SELECT est utilisée pour récupérer des données dans une ou plusieurs tables d'une base de données. C'est la commande SQL la plus utilisée et elle constitue la base de presque toutes les requêtes de base de données.

🧱 Syntaxe SELECT

SELECT column1, column2
FROM table_name
WHERE condition;
  • SELECT - spécifie les colonnes que vous voulez récupérer
  • FROM - définit la table à interroger
  • WHERE - (optionnel) filtre les lignes sur la base de conditions

🔠 Sélectionner des colonnes spécifiques

Vous pouvez récupérer une ou plusieurs colonnes en spécifiant leur nom :

SELECT name, email
FROM users;

Cette commande ne renvoie que les champs name et email de la table users.

🌐 Sélectionner toutes les colonnes

Pour récupérer toutes les colonnes, utilisez le caractère générique * :

SELECT *
FROM users;

Bien que cette méthode soit rapide et pratique, il est souvent préférable de ne sélectionner que les colonnes dont vous avez besoin, en particulier pour des raisons de performance.

⚙️ Utilisation d'alias (AS)

Vous pouvez renommer les colonnes dans votre sortie en utilisant des aliases avec le mot-clé AS :

SELECT name AS full_name, email AS contact_email
FROM users;

Cela ne renomme pas la colonne dans le tableau, mais modifie uniquement la façon dont elle est affichée dans le résultat.

📌 Exemples concrets

Exemple 1 - Obtenir tous les noms et prix des produits :

SELECT product_name, price
FROM products;

Exemple 2 - Filtrer les utilisateurs par pays :

SELECT name, country
FROM users
WHERE country = 'Canada';

✅ Conseils

  • Commencez toujours par SELECT et faites suivre de FROM.
  • Utiliser des virgules pour séparer les noms de colonnes
  • Éviter le SELECT * en production sauf en cas d'absolue nécessité
  • Utiliser des alias pour faciliter la lecture des requêtes complexes

📘 Recap

  • L'instruction SELECT est utilisée pour extraire des données d'une base de données.
  • Vous pouvez sélectionner des colonnes spécifiques ou utiliser * pour les obtenir toutes.
  • Utilisez des alias pour une meilleure lisibilité.
  • C'est la base de toute interrogation de données en SQL.
