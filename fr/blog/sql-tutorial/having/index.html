🎯 Qu'est-ce que l'AVOIR ?

La clause HAVING est utilisée pour filtrer les groupes créés par la clause GROUP BY.

Alors que la clause WHERE filtre les lignes individuelles, HAVING vous permet de filtrer le résultat de données agrégées.

🧱 Syntaxe HAVING

SELECT column, AGGREGATE_FUNCTION(column)
FROM table
GROUP BY column
HAVING AGGREGATE_FUNCTION(column) condition;

🧪 Exemple : Produits dont les ventes totales sont supérieures à un seuil

SELECT product_id, SUM(amount) AS total_sales
FROM sales
GROUP BY product_id
HAVING SUM(amount) > 1000;

Seuls les produits dont les ventes totales sont supérieures à 1000 sont renvoyés.

🧪 Exemple : Clients ayant passé plus de 3 commandes

SELECT customer_id, COUNT(*) AS order_count
FROM orders
GROUP BY customer_id
HAVING COUNT(*) > 3;

Seuls les clients ayant passé plus de trois commandes seront renvoyés.

⚖️ HAVING vs WHERE

Caractéristiques Où HAVAGE

Travaille sur Lignes Groupes (données agrégées)

📘 Recap

  • Utiliser HAVING pour filtrer les données après qu'elles aient été groupées
  • Il fonctionne avec les fonctions d'agrégation comme COUNT(), SUM(), et AVG()
  • Contrairement à WHERE, il est utilisé pour filtrer les résultats d'un GROUP BY
