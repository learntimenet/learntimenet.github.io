🔍 Qu'est-ce que l'IN ?

L'opérateur IN vous permet de filtrer les enregistrements sur la base d'une liste de valeurs.

C'est une alternative plus propre à l'utilisation de plusieurs conditions OR dans une clause WHERE.

🧱 Syntaxe de base

SELECT * FROM table_name
WHERE column_name IN (value1, value2, value3);

🧪 Exemple : Faire correspondre plusieurs pays

SELECT * FROM users
WHERE country IN ('USA', 'Canada', 'Mexico');

✅ Équivalent à :

WHERE country = 'USA' OR country = 'Canada' OR country = 'Mexico'

🧪 Exemple : Avec des nombres

Trouver des commandes avec des statuts spécifiques

SELECT * FROM orders
WHERE status_id IN (1, 3, 5);

📊 Exemple : Avec une sous-requête

Obtenir les utilisateurs qui ont passé une commande

SELECT * FROM users
WHERE user_id IN (
  SELECT user_id FROM orders
);

La sous-requête renvoie une liste utilisée comme filtre dans la requête principale.

⚠️ Notes

  • Permet d'obtenir des requêtes plus propres et plus lisibles.
  • Les valeurs de la liste doivent être du même type de données
  • Pour des raisons de performance, évitez les listes très volumineuses à l'intérieur de IN.

📘 Recap

  • IN filtre les enregistrements en recherchant n'importe laquelle d'une liste de valeurs
  • Peut être utilisé avec du texte, des nombres ou des sous-requêtes.
  • C'est une meilleure alternative aux longues chaînes de conditions "OU".
