🌐 Qu'est-ce que le FULL JOIN ?

Un FULL JOIN (ou FULL OUTER JOIN) renvoie :

  • Tous les enregistrements de la table gauche
  • Tous les enregistrements de la table de droite
  • 🔄 Des allumettes lorsqu'elles sont disponibles
  • ❌ NULL lorsqu'il n'y a pas de correspondance d'un côté ou de l'autre

C'est comme combiner LEFT JOIN et RIGHT JOIN.

🧱 FULL JOIN Syntaxe

SELECT columns
FROM table1
FULL JOIN table2
ON table1.column = table2.column;

🧪 Exemple : Employés et départements

SELECT employees.name, departments.dept_name
FROM employees
FULL JOIN departments
ON employees.dept_id = departments.id;

Cela reviendra :

  • Employés avec ou sans département
  • Départements avec ou sans employés

🧪 Exemple : Commandes et expéditions

SELECT orders.id, shipments.tracking_number
FROM orders
FULL JOIN shipments
ON orders.id = shipments.order_id;

Parfait pour le repérage :

  • Commandes non expédiées
  • Envois sans référence de commande valide

🔍 Quand utiliser FULL JOIN ?

Utilisez-le quand vous le souhaitez :

  • Une vue complète des deux tableaux
  • Pour détecter les incohérences ou les liens manquants
  • Pour combiner la logique de jointure à gauche et à droite en une seule requête

📘 Recap

  • FULL JOIN montre tout des deux côtés
  • Affiche NULL lorsqu'il n'existe pas d'enregistrement correspondant.
  • Idéal pour les requêtes de rapprochement et d'audit
