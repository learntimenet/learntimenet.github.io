🧭 Qu'est-ce que la jonction à gauche ?

LEFT JOIN retourne :

  • Toutes les lignes du tableau de gauche.
  • ✅ Les lignes correspondantes du tableau de droite
  • ❌ NULL s'il n'y a aucune correspondance à droite

C'est utile lorsque vous voulez conserver toutes les données de la première table, même s'il n'y a rien de lié dans la seconde.

🧱 LEFT JOIN Syntaxe

SELECT columns
FROM table1
LEFT JOIN table2
ON table1.column = table2.column;

🧪 Exemple : Utilisateurs et commandes

SELECT users.name, orders.amount
FROM users
LEFT JOIN orders
ON users.id = orders.user_id;

Cette requête affichera tous les utilisateurs, même ceux qui n'ont pas passé de commande.
Les utilisateurs sans commandes afficheront NULL pour orders.amount.

🧪 Exemple : Produits et commentaires

SELECT products.name, reviews.rating
FROM products
LEFT JOIN reviews
ON products.id = reviews.product_id;

Utile pour lister tous les produits, qu'ils aient ou non fait l'objet d'une évaluation.

⚖️ INNER vs LEFT JOIN

Caractéristiques INNER JOIN LEFT JOIN
Retourne les matchs ✅ Oui ✅ Oui
Conserve les non-concordances ❌ Non ✅ De la table de gauche

📘 Recap

  • LEFT JOIN inclut toutes les lignes de la table de gauche, avec les correspondances (ou NULLs) de la droite.
  • Idéal pour présenter "tous les articles, même s'ils n'ont pas été utilisés".
  • Permet d'identifier les données manquantes ou non concordantes
