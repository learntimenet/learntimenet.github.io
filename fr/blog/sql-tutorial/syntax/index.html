📐 Structure des instructions SQL de base

Le langage SQL suit une syntaxe simple, facile à lire et à écrire.

Voici le format de base :

SELECT column1, column2
FROM table_name
WHERE condition;

Chaque commande SQL comporte généralement les éléments suivants

  1. Un mot-clé (comme SELECT, INSERT, UPDATE, etc.)
  2. Clauses (comme FROM, WHERE, ORDER BY, etc.)
  3. Conditions ou valeurs (pour filtrer ou modifier les données)

🧱 Mots-clés SQL courants

Les mots-clés SQL sont des mots réservés qui indiquent à la base de données l'action à effectuer.

Voici quelques mots-clés SQL couramment utilisés :

. . . . .
Mot clé Description
SELECT Récupère les données d'un tableau
FROM Spécifie la table
Où Filtre les lignes en fonction d'une condition
INSERT INTO Ajoute de nouvelles lignes à un tableau
MISE A JOUR Modifie les données existantes
DELETE Supprime des lignes d'un tableau
CREER UNE TABLE Crée une nouvelle table
ALTER TABLE Modifie la structure d'une table
DROP TABLE Supprime complètement une table

🧠 Conseil: Même si les mots-clés SQL ne sont pas sensibles à la casse, les écrire en majuscules améliore la lisibilité.

🔤 Sensibilité à la casse en SQL

  • Les mots clés comme SELECT, FROM, et WHERE ne sont pas sensibles à la casse.
  • Les noms des tables et des colonnes** peuvent être sensibles à la casse, en fonction du système de base de données (par exemple, MySQL sous Linux = sensible à la casse).
SELECT Name FROM Employees;
-- Same as
select name from employees;

Meilleure pratique : Utilisez des majuscules pour les mots-clés SQL et des minuscules ou des majuscules pour les noms de tables/colonnes.

🧾 Avez-vous besoin d'un point-virgule ?

Oui ! Le semicolon (;) est utilisé pour terminer une instruction SQL, en particulier lors de l'écriture de plusieurs requêtes dans un script.

SELECT * FROM users;
DELETE FROM users WHERE id = 5;

Certains environnements SQL peuvent ne pas l'exiger pour les requêtes simples, mais il est bon de toujours l'inclure.

✨ Meilleures pratiques de formatage

Pour rendre votre SQL lisible et professionnel :

  • ✅ Utiliser des majuscules pour les mots-clés
  • ✅ Diviser les longues requêtes en plusieurs lignes
  • ✅ Utiliser l'indentation pour montrer la hiérarchie
  • ✅ Commentez vos questions si nécessaire

Exemple :

-- Get active users who signed up this month
SELECT id, name, signup_date
FROM users
WHERE active = 1
  AND MONTH(signup_date) = MONTH(CURRENT_DATE);

📘 Recap

  • Le langage SQL est lisible et structuré, avec une syntaxe de base comprenant des mots-clés, des clauses et des conditions.
  • Les mots-clés ne sont pas sensibles à la casse, mais les conventions d'appellation sont importantes.
  • Pour des raisons de cohérence, les déclarations doivent toujours se terminer par un point-virgule.
  • Une mise en forme propre améliore la lisibilité et la maintenabilité de votre code.
