🧩 Qu'est-ce que INSERT INTO ?

L'instruction INSERT INTO est utilisée pour ajouter de nouveaux enregistrements à une table de base de données.

Il vous permet d'alimenter votre tableau avec des données fraîches, soit une ligne à la fois, soit plusieurs lignes à la fois.

🧱 Syntaxe de base

Insérer une seule ligne (en spécifiant les colonnes) :

INSERT INTO table_name (column1, column2, column3)
VALUES (value1, value2, value3);

Insérer plusieurs lignes :

INSERT INTO table_name (column1, column2)
VALUES
  (value1a, value2a),
  (value1b, value2b);

🧪 Exemple : Ajout de nouveaux utilisateurs

Utilisateur unique

INSERT INTO users (name, email, country)
VALUES ('Alice', 'alice@example.com', 'USA');

Utilisateurs multiples

INSERT INTO users (name, email, country)
VALUES
  ('Bob', 'bob@example.com', 'UK'),
  ('Lena', 'lena@example.com', 'Canada');

⚠️ Conseils et pièges courants

  • 🔒 Toujours spécifier les noms des colonnes afin d'éviter les problèmes d'incompatibilité d'ordre.
  • Les types de données doivent correspondre aux définitions des colonnes (par exemple, pas de texte dans les champs numériques).
  • 🚫 Les champs NOT NULL manquants sans valeurs par défaut provoqueront des erreurs
  • 🔄 Utiliser RETURNING * (PostgreSQL) pour récupérer les données insérées

📘 Recap

  • INSERT INTO ajoute des données à une table
  • Vous pouvez insérer une ou plusieurs lignes à la fois
  • Toujours faire correspondre les types de valeurs aux types de colonnes
  • Nommer explicitement les colonnes est une bonne pratique
