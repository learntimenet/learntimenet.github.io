🆔 Que sont les alias SQL ?

Les alias vous permettent d'attribuer un nom temporaire à une colonne ou à une table en utilisant le mot-clé AS.
Ils sont souvent utilisés pour :

  • Simplifier les noms de colonnes complexes
  • Renommer les colonnes de sortie
  • Améliorer la lisibilité des requêtes
  • Rendre les jointures et les sous-requêtes plus faciles à suivre

Les alias n'existent que pendant l'exécution de la requête - ils ne modifient pas les noms des tables/colonnes.

🧱 Syntaxe de l'alias de colonne

SELECT column_name AS alias_name
FROM table_name;

🧱 Syntaxe des alias de table

SELECT t.column_name
FROM table_name AS t;

Le mot-clé AS est optionnel - SELECT nom_colonne nom_alias fonctionne également.

🧪 Exemple : Renommer une colonne

SELECT first_name AS name
FROM employees;

🧪 Exemple : Alias de table

SELECT u.name, o.amount
FROM users AS u
JOIN orders AS o ON u.id = o.user_id;

🧪 Exemple : Alias dans le calcul

SELECT price * quantity AS total
FROM sales;

✨ Pourquoi utiliser des alias ?

  • 🧼 Nettoyer les noms des colonnes de sortie
  • 🧠 Meilleure compréhension des requêtes complexes
  • 🔁 Références plus courtes dans les JOIN et les sous-requêtes
  • 🎯 Améliore la lisibilité du code dans les rapports

📘 Recap

  • Utilisez AS pour renommer temporairement des colonnes ou des tables.
  • Les alias permettent de simplifier et de clarifier la sortie SQL
  • Idéal pour les jointures, les calculs et les requêtes complexes
