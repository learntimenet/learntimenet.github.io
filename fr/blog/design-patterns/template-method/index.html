🔍 Qu'est-ce que le modèle de la méthode des gabarits ?

Le Template Method Pattern définit le skeleton d'un algorithme dans une méthode, en reportant certaines étapes à des sous-classes.
Les sous-classes peuvent redéfinir certaines étapes d'un algorithme sans en modifier la structure.

Pensez à faire du thé ou du café : les étapes sont similaires (faire bouillir l'eau, infuser, verser, ajouter des condiments), mais certaines étapes varient légèrement.

🧩 Quand l'utiliser ?

Vous devez utiliser le modèle de la méthode des modèles dans les cas suivants :

  • Vous avez plusieurs classes avec des algorithmes similaires mais quelques différences dans les étapes.
  • Vous voulez éviter la duplication du code
  • Vous avez besoin d'une structure d'algorithme cohérente, mais d'étapes individuelles flexibles.

🧱 Implémentation Java

👩‍🏫 Étape 1 : Définir la classe abstraite

public abstract class Beverage {
    
    // Template method
    public final void prepareRecipe() {
        boilWater();
        brew();
        pourInCup();
        addCondiments();
    }

    abstract void brew();
    abstract void addCondiments();

    public void boilWater() {
        System.out.println("Boiling water...");
    }

    public void pourInCup() {
        System.out.println("Pouring into cup...");
    }
}

☕ Étape 2 : Implémentations concrètes

public class Tea extends Beverage {

    void brew() {
        System.out.println("Steeping the tea...");
    }

    void addCondiments() {
        System.out.println("Adding lemon...");
    }
}

public class Coffee extends Beverage {

    void brew() {
        System.out.println("Dripping coffee through filter...");
    }

    void addCondiments() {
        System.out.println("Adding sugar and milk...");
    }
}

✅ Exemple d'utilisation

public class TemplateMethodPatternDemo {
    public static void main(String[] args) {
        Beverage tea = new Tea();
        tea.prepareRecipe();

        System.out.println();

        Beverage coffee = new Coffee();
        coffee.prepareRecipe();
    }
}

🌐 Analogie avec le monde réel

  • Faire du thé et du café : Même séquence, mais les étapes "infuser" et "ajouter des condiments" diffèrent.
  • Recettes de cuisine : Faites un gâteau, mais les ingrédients et les arômes diffèrent.
  • Génération de documents : Étapes communes avec de légères variations pour les différents types de documents.

⚠️ Les pièges à éviter

  • rendre la classe de base trop rigide (les sous-classes doivent être flexibles)
  • ❌ Utilisation d'un modèle de méthode là où un simple héritage ou une interface suffirait
  • ❌ Laisser les sous-classes violer la séquence - le squelette doit être cohérent

✅ Meilleures pratiques

  • Utilisez final sur la méthode template pour éviter de la surcharger
  • Garder les méthodes abstraites claires et minimales
  • Combiner avec les crochets (étapes facultatives) si nécessaire pour plus de flexibilité.

📘 Recap

  • Le modèle de méthode définit le squelette d'un algorithme, avec des étapes personnalisables.
  • Favorise la réutilisation du code et les structures cohérentes.
  • Idéal pour les recettes, les flux de travail, le traitement des documents, etc.
