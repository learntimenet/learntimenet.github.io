🔍 Quel est le schéma d'adaptation ?

Le modèle d'adaptateur permet à des classes incompatibles de travailler ensemble en transformant une interface en une autre.

Pensez-y comme à un adaptateur de prise de courant - il connecte deux systèmes mal assortis afin qu'ils puissent communiquer.

🔁 Quand l'utiliser

Utilisez le modèle d'adaptateur lorsque :

  • Vous devez utiliser une classe existante mais son interface ne correspond pas à ce dont vous avez besoin
  • Vous voulez intégrer le code ancien dans un nouveau système
  • Vous construisez un système réutilisable qui nécessite une flexibilité d'interface.

🧱 Implémentation Java

🧩 Scénario

Vous avez une interface MediaPlayer qui s'attend à lire des fichiers audio, mais un AdvancedMediaPlayer tiers supporte des méthodes différentes.

🎧 Interfaces et classes

// Target interface
public interface MediaPlayer {
    void play(String audioType, String fileName);
}


// Adaptee (incompatible class)
public class AdvancedMediaPlayer {
    public void playMp4(String fileName) {
        System.out.println("Playing MP4 file: " + fileName);
    }

    public void playVlc(String fileName) {
        System.out.println("Playing VLC file: " + fileName);
    }
}

🔌 Classe d'adaptateur

public class MediaAdapter implements MediaPlayer {
    private AdvancedMediaPlayer advancedPlayer = new AdvancedMediaPlayer();

    public void play(String audioType, String fileName) {
        if(audioType.equalsIgnoreCase("vlc")) {
            advancedPlayer.playVlc(fileName);
        } else if(audioType.equalsIgnoreCase("mp4")) {
            advancedPlayer.playMp4(fileName);
        } else {
            System.out.println("Invalid format: " + audioType);
        }
    }
}

✅ Usage

public class AudioPlayer implements MediaPlayer {
    private MediaAdapter adapter;

    public void play(String audioType, String fileName) {
        if(audioType.equalsIgnoreCase("mp3")) {
            System.out.println("Playing MP3 file: " + fileName);
        } else {
            adapter = new MediaAdapter();
            adapter.play(audioType, fileName);
        }
    }
}


AudioPlayer player = new AudioPlayer();
player.play("mp3", "song.mp3");
player.play("mp4", "video.mp4");
player.play("vlc", "movie.vlc");

🔌 Analogie avec le monde réel

Imaginez que vous voyagiez à l'étranger. La fiche de votre ordinateur portable n'est pas adaptée à la prise de courant - vous utilisez donc un adaptateur de voyage pour combler l'écart. Même appareil, compatible grâce à la conversion.

⚠️ Les pièges à éviter

  • ❌ Création d'adaptateurs trop complexes pour des intégrations très simples
  • ❌ Ignorer le coût de performance de l'adaptation au moment de l'exécution
  • ❌ Confusion avec le motif Decorator ou Facade.

✅ Meilleures pratiques

  • Garder une logique d'adaptation simple et ciblée
  • Préférer la composition à l'héritage
  • Utiliser les adaptateurs pour la compatibilité d'interface, pas pour les changements de comportement

📘 Recap

  • Le modèle de l'adaptateur permet à des classes ayant des interfaces incompatibles de travailler ensemble.
  • Il enveloppe une classe dans une autre pour traduire les interfaces
  • Très utile pour l'intégration de systèmes tiers ou patrimoniaux
  • Analogie avec le monde réel : adaptateurs de prise, clés USB, traducteurs linguistiques
