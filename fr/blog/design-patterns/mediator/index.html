🔍 Qu'est-ce que le modèle du médiateur ?

Le Mediator Pattern définit un objet qui centralise la communication entre d'autres objets. Au lieu de se référer directement l'un à l'autre, les objets interagissent par l'intermédiaire du médiateur.

Imaginez une tour de contrôle du trafic aérien : elle coordonne les avions au lieu que les avions communiquent directement entre eux.

🧩 Quand l'utiliser ?

Vous devez utiliser le modèle du médiateur dans les cas suivants :

  • Les objets sont trop étroitement couplés et vous voulez réduire les dépendances directes
  • Vous avez une communication complexe entre plusieurs classes
  • Vous souhaitez centraliser la logique de contrôle et de communication

🧱 Implémentation Java

👩‍🏫 Étape 1 : Définir l'interface du médiateur

public interface ChatMediator {
    void sendMessage(String message, User user);
    void addUser(User user);
}

🧱 Étape 2 : Médiateur concret

import java.util.ArrayList;
import java.util.List;

public class ChatRoomMediator implements ChatMediator {
    private List<User> users = new ArrayList<>();

    public void addUser(User user) {
        users.add(user);
    }

    public void sendMessage(String message, User sender) {
        for (User user : users) {
            // Don't send message to the sender
            if (user != sender) {
                user.receive(message);
            }
        }
    }
}

👤 Étape 3 : Définir la classe abstraite de l'utilisateur

public abstract class User {
    protected ChatMediator mediator;
    protected String name;

    public User(ChatMediator mediator, String name) {
        this.mediator = mediator;
        this.name = name;
    }

    public abstract void send(String message);
    public abstract void receive(String message);
}

🙋‍♂️ Étape 4 : Mise en œuvre concrète par l'utilisateur

public class ChatUser extends User {

    public ChatUser(ChatMediator mediator, String name) {
        super(mediator, name);
    }

    public void send(String message) {
        System.out.println(this.name + " sends: " + message);
        mediator.sendMessage(message, this);
    }

    public void receive(String message) {
        System.out.println(this.name + " receives: " + message);
    }
}

✅ Exemple d'utilisation

public class MediatorPatternDemo {
    public static void main(String[] args) {
        ChatMediator chatRoom = new ChatRoomMediator();

        User user1 = new ChatUser(chatRoom, "Alice");
        User user2 = new ChatUser(chatRoom, "Bob");
        User user3 = new ChatUser(chatRoom, "Charlie");

        chatRoom.addUser(user1);
        chatRoom.addUser(user2);
        chatRoom.addUser(user3);

        user1.send("Hi everyone!");
    }
}

🌐 Analogie avec le monde réel

  • Contrôle du trafic aérien : Les pilotes communiquent avec une tour de contrôle plutôt que directement entre eux.
  • Applications de chat de groupe : Les messages sont acheminés via un serveur central.
  • Systèmes domestiques intelligents : Les appareils sont reliés à un concentrateur plutôt que directement les uns aux autres.

⚠️ Les pièges à éviter

  • ❌ Surcharge du médiateur : il peut devenir trop complexe s'il gère trop de logique
  • ❌ Créer des médiateurs inutiles pour des communications simples
  • ❌ Couplage étroit avec un médiateur spécifique au lieu d'utiliser une interface

✅ Meilleures pratiques

  • Garder le médiateur concentré et simple
  • Abstraction du médiateur par le biais d'une interface pour plus de flexibilité
  • Refondre la logique de communication graduellement au fur et à mesure que les systèmes deviennent plus complexes

📘 Recap

  • Le modèle du médiateur centralise la communication pour réduire les dépendances entre les classes
  • Il favorise le couplage souple et améliore la maintenabilité.
  • Mieux adapté pour les salons de discussion, le contrôle du trafic aérien et les systèmes intelligents
