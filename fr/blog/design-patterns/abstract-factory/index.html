🔍 Qu'est-ce que le modèle d'usine abstraite ?

Le Modèle d'usine abstraite fournit une interface pour créer des familles d'objets apparentés sans spécifier leurs classes concrètes.

C'est comme une usine d'usines.

🏭 Usine vs Usine abstraite

Aspect Méthode d'usine Fabrique abstraite
Focus Un objet à la fois Multiples objets apparentés
Structure Produit unique Familles de produits
Exemple NotificationFactory UIComponentFactory

🧱 Exemple de mise en œuvre en Java

Imaginez que vous construisiez une interface utilisateur pour plusieurs systèmes d'exploitation - vous voulez produire des boutons et des cases à cocher cohérents pour chacun d'entre eux.

🧩 Étape 1 : Définir les interfaces des produits

public interface Button {
    void render();
}

public interface Checkbox {
    void render();
}

🧩 Étape 2 : Implémentations concrètes

public class WindowsButton implements Button {
    public void render() {
        System.out.println("Rendering Windows Button");
    }
}

public class MacButton implements Button {
    public void render() {
        System.out.println("Rendering Mac Button");
    }
}

public class WindowsCheckbox implements Checkbox {
    public void render() {
        System.out.println("Rendering Windows Checkbox");
    }
}

public class MacCheckbox implements Checkbox {
    public void render() {
        System.out.println("Rendering Mac Checkbox");
    }
}

🧩 Étape 3 : Créer une usine abstraite

public interface GUIFactory {
    Button createButton();
    Checkbox createCheckbox();
}

🧩 Étape 4 : Usines de béton

public class WindowsFactory implements GUIFactory {
    public Button createButton() {
        return new WindowsButton();
    }
    public Checkbox createCheckbox() {
        return new WindowsCheckbox();
    }
}

public class MacFactory implements GUIFactory {
    public Button createButton() {
        return new MacButton();
    }
    public Checkbox createCheckbox() {
        return new MacCheckbox();
    }
}

✅ Usage

public class Application {
    private Button button;
    private Checkbox checkbox;

    public Application(GUIFactory factory) {
        button = factory.createButton();
        checkbox = factory.createCheckbox();
    }

    public void renderUI() {
        button.render();
        checkbox.render();
    }
}


GUIFactory factory = new MacFactory();
Application app = new Application(factory);
app.renderUI();

💡 Analogie avec le monde réel

Pensez à une salle d'exposition de meubles. Une usine de meubles abstraite peut produire une chaise moderne et une table moderne, ou une chaise victorienne et une table victorienne - toujours un ensemble assorti.

⚠️ Les pièges à éviter

  • ❌ Complication excessive des scénarios de création d'objets simples
  • ❌ Couplage étroit avec les usines - favoriser les interfaces
  • ❌ Rendre toutes les familles de produits identiques

✅ Meilleures pratiques

  • Favoriser les interfaces pour tous les types de produits
  • Garder les usines extensibles
  • Combiner avec l'injection de dépendance lorsque c'est possible
  • Maintenir une séparation claire entre le code client et la logique de création.

📘 Recap

  • La fabrique abstraite vous permet de produire des familles d'objets apparentées
  • Il permet d'isoler la création d'un objet de son utilisation.
  • Idéal pour les cadres d'interface utilisateur, les plugins et les systèmes de thématisation
  • Favorise l'évolutivité et la cohérence des produits
