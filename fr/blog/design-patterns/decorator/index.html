🔍 Qu'est-ce que le motif décoratif ?

Le modèle du décorateur vous permet d'attacher de nouveaux comportements aux objets au moment de l'exécution en les plaçant à l'intérieur d'objets enveloppants, ou "décorateurs", qui implémentent la même interface.

Il s'agit d'une alternative flexible à la sous-classe pour étendre les fonctionnalités.

🧩 Quand l'utiliser ?

Utilisez le modèle du décorateur dans les cas suivants

  • Vous devez ajouter des fonctionnalités aux objets sans modifier leur classe.
  • Vous voulez éviter de créer trop de sous-classes
  • Vous voulez superposer des comportements de manière flexible et dynamique

🧱 Implémentation Java

🧩 Étape 1 : Définir l'interface du composant

public interface Coffee {
    String getDescription();
    double getCost();
}

☕ Étape 2 : Créer un composant concret

public class BasicCoffee implements Coffee {
    public String getDescription() {
        return "Basic Coffee";
    }

    public double getCost() {
        return 2.0;
    }
}

🧥 Étape 3 : Décorateur abstrait

public abstract class CoffeeDecorator implements Coffee {
    protected Coffee decoratedCoffee;

    public CoffeeDecorator(Coffee coffee) {
        this.decoratedCoffee = coffee;
    }

    public String getDescription() {
        return decoratedCoffee.getDescription();
    }

    public double getCost() {
        return decoratedCoffee.getCost();
    }
}

🍫 Étape 4 : Décorateurs de béton

public class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }

    public String getDescription() {
        return super.getDescription() + ", Milk";
    }

    public double getCost() {
        return super.getCost() + 0.5;
    }
}


public class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) {
        super(coffee);
    }

    public String getDescription() {
        return super.getDescription() + ", Sugar";
    }

    public double getCost() {
        return super.getCost() + 0.2;
    }
}

✅ Usage

Coffee coffee = new BasicCoffee();
coffee = new MilkDecorator(coffee);
coffee = new SugarDecorator(coffee);

System.out.println(coffee.getDescription()); // Basic Coffee, Milk, Sugar
System.out.println("Total: __HTML__quot; + coffee.getCost()); // Total: $2.7

☕ Analogie avec le monde réel

Imaginez que vous commandiez un café : vous commencez par une tasse de base, puis vous la décorez avec du lait, du sucre, de la crème fouettée, etc. Chaque ajout enveloppe le café et ajoute sa propre saveur (et son propre prix) !

⚠️ Les pièges à éviter

  • ❌ Rendre les décorateurs trop complexes - garder chacun d'eux petit et ciblé
  • ❌ Oublier de déléguer des méthodes au composant enveloppé
  • ❌ Sur-utilisation alors que la sous-classification serait plus simple

✅ Meilleures pratiques

  • Garder les décorateurs légers et à usage unique
  • S'en tenir au principe ouvert/fermé : ouvert à l'extension, fermé à la modification
  • Décorateurs en chaîne dans l'ordre dans lequel ils doivent s'appliquer

📘 Recap

  • Le modèle du décorateur permet une extension dynamique du comportement de l'objet.
  • Une alternative utile aux arbres d'héritage profonds
  • Idéal pour les composants d'interface utilisateur, les flux d'entrée/sortie et les bascules de fonctions.
  • Facile à combiner avec de nouvelles fonctionnalités
