🔍 Qu'est-ce que le modèle de l'itérateur ?

Le modèle d'itérateur permet d'accéder aux éléments d'une collection de manière séquentielle sans exposer sa représentation sous-jacente.

Pensez à une liste de lecture : vous pouvez vous déplacer chanson par chanson sans savoir comment les chansons sont stockées en interne.

🧩 Quand l'utiliser ?

Vous devez utiliser le modèle de l'itérateur dans les cas suivants :

  • Vous devez traverser une collection sans révéler sa structure
  • Vous voulez plusieurs traversées en même temps
  • Vous voulez une manière standardisée de faire des boucles dans différentes structures de données.

🧱 Implémentation Java

👩‍🏫 Étape 1 : Définir l'interface de l'itérateur

public interface Iterator<T> {
    boolean hasNext();
    T next();
}

🏛️ Étape 2 : Définir l'interface de collecte

public interface Collection<T> {
    Iterator<T> createIterator();
}

🎵 Étape 3 : Collecte du béton

import java.util.ArrayList;
import java.util.List;

public class Playlist implements Collection<String> {
    private List<String> songs = new ArrayList<>();

    public void addSong(String song) {
        songs.add(song);
    }

    public Iterator<String> createIterator() {
        return new PlaylistIterator(songs);
    }
}

📜 Étape 4 : Itérateur concret

import java.util.List;

public class PlaylistIterator implements Iterator<String> {
    private List<String> songs;
    private int position = 0;

    public PlaylistIterator(List<String> songs) {
        this.songs = songs;
    }

    public boolean hasNext() {
        return position < songs.size();
    }

    public String next() {
        return songs.get(position++);
    }
}

✅ Exemple d'utilisation

public class IteratorPatternDemo {
    public static void main(String[] args) {
        Playlist playlist = new Playlist();
        playlist.addSong("Song A");
        playlist.addSong("Song B");
        playlist.addSong("Song C");

        Iterator<String> iterator = playlist.createIterator();

        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}

🌐 Analogie avec le monde réel

  • Listes de lecture musicales** : Navigation dans la chanson suivante, la chanson précédente
  • Les étagères** : Numériser chaque livre un par un
  • Galeries de photos** : Faites défiler les photos de manière séquentielle

⚠️ Les pièges à éviter

  • ❌ Modifier la collection pendant l'itération peut conduire à des erreurs de modification simultanée.
  • ❌ Ne pas gérer correctement la fin de la collecte (vérifications hasNext manquantes)
  • ❌ Création d'itérateurs personnalisés trop complexes alors que Java fournit des itérateurs intégrés (Iterator, Iterable)

✅ Meilleures pratiques

  • Utilisez toujours hasNext() avant d'appeler next()
  • Préférer l'utilisation de boucles for-loops améliorées (for-each) lorsque c'est possible dans les cas simples
  • Garder les itérateurs simples et ciblés

📘 Recap

  • Le motif de l'itérateur vous permet de traverser des collections sans en connaître la structure interne.
  • Favorise un libre couplage entre la logique de collecte et la logique de traversée
  • Idéal pour les collections, les listes de lecture, les étagères, etc.
