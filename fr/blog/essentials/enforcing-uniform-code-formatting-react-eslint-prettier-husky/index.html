Enforcer le formatage en code uniforme dans React avec ESLint, Prettier et Husky

Le maintien d'un style et d'une qualité de code cohérents est crucial dans tout projet React, en particulier lorsque votre équipe ou votre base de code s'agrandit. Cet article vous guidera dans la configuration de ESLint, Prettier, et Husky pour votre projet React + TypeScript avec une configuration minimale et efficace pour appliquer un formatage et un linting uniformes.

Pourquoi imposer un formatage uniforme ?

  • Améliore la lisibilité** en appliquant un style cohérent à l'ensemble de votre base de code.
  • Réduit les frictions lors de l'examen du code en évitant les débats sur le style
  • Prévient les bogues et les erreurs en amont grâce au linting
  • Automatise le formatage pour faire gagner du temps aux développeurs

Aperçu des outils

  • Prettier : Un formateur de code qui formate automatiquement vos fichiers.
  • ESLint : Un outil de linting qui détecte les erreurs de code et applique les normes de codage.
  • Husky : Un gestionnaire de hooks Git pour exécuter des commandes comme le linting et le formatage avant les commits.
  • lint-staged : Exécute les linters uniquement sur les fichiers mis en scène pour des vérifications plus rapides et plus ciblées.

Étape 1 : Installer les dépendances nécessaires

Exécutez cette commande à la racine de votre projet pour installer tous les dépendances nécessaires :

npm install --save-dev prettier eslint eslint-config-prettier @typescript-eslint/parser @typescript-eslint/eslint-plugin husky lint-staged

Il s'agit notamment de

  • prettier pour le formatage
  • eslint pour les fonctionnalités de base de linting
  • @typescript-eslint/parser pour permettre à ESLint d'analyser la syntaxe TypeScript (interfaces, types, enums, etc.)
  • @typescript-eslint/eslint-plugin pour ajouter des règles de linting et des meilleures pratiques spécifiques à TypeScript
  • eslint-config-prettier pour désactiver les règles ESLint qui pourraient entrer en conflit avec le formatage de Prettier
  • husky et lint-staged pour gérer les hooks de pré-commission Git afin d'exécuter automatiquement les linters et les formateurs.

Étape 2 : Configurer Prettier

Créez un fichier .prettierrc à la racine de votre projet avec le contenu suivant :

{
  "semi": true,
  "singleQuote": true,
  "printWidth": 80,
  "trailingComma": "all",
  "endOfLine": "lf"
}

Elle définit les règles de formatage de base, telles que l'utilisation de points-virgules, de guillemets simples et de retours à la ligne cohérents.

Étape 3 : Configurer ESLint

Créez un fichier .eslintrc.json à la racine avec cette configuration minimale axée sur TypeScript :

{
  "env": {
    "browser": true,
    "es2021": true
  },
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended",
    "prettier"
  ],
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 12,
    "sourceType": "module"
  },
  "plugins": ["@typescript-eslint"],
  "rules": {}
}

Cela permet d'appliquer les règles ESLint et TypeScript recommandées tout en reportant les problèmes de formatage sur Prettier.

Étape 4 : Configuration de lint-staged et de Husky

Ajoutez ceci à votre package.json pour indiquer à lint-staged quels fichiers doivent être vérifiés et corrigés lors du commit :

"lint-staged": {
  "src/**/*.{js,jsx,ts,tsx}": [
    "eslint --fix",
    "prettier --write"
  ]
}

Initialiser Husky et ajouter un crochet de pré-commission :

npx husky install
npx husky add .husky/pre-commit "npx lint-staged"

Cela permet d'exécuter ESLint et Prettier sur vos fichiers stagés avant chaque livraison, ce qui aide à maintenir votre base de code propre automatiquement.

Étape 5 : (Facultatif) Configurer votre éditeur pour le formatage lors de l'enregistrement

Si vous utilisez VS Code, installez les extensions Prettier et ESLint et ajoutez les éléments suivants à votre espace de travail ou aux paramètres de l'utilisateur pour formater et résoudre les problèmes de lint lors de l'enregistrement :

{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}

Améliorations futures

  • Ajouter des règles de linting spécifiques à React en installant eslint-plugin-react et eslint-plugin-react-hooks.
  • Intégration de stylelint pour l'analyse CSS/Tailwind.
  • Ajout de règles de test (par exemple, pour Jest).

Conclusion

Avec seulement quelques dépendances et fichiers de configuration, vous pouvez automatiser et appliquer un formatage de code et un linting cohérents dans vos projets React + TypeScript. Cela permet aux développeurs de gagner du temps, de réduire les erreurs et d'améliorer la collaboration - une étape essentielle vers une base de code professionnelle.
