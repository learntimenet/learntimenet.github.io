📖 Qu'est-ce que le principe de substitution de Liskov ?

Défini par l'informaticienne Barbara Liskov, ce principe est le suivant :

Les objets d'une superclasse doivent pouvoir être remplacés par des objets d'une sous-classe sans altérer la correction du programme.

En bref : Si la classe S est une sous-classe de la classe T, alors les objets de type T peuvent être remplacés par des objets de type S sans casser le programme.

🧠 Pourquoi le FSL est important

LSP garantit que l'héritage est logique et sûr. Si une sous-classe surcharge une méthode ou modifie le comportement attendu, cela peut conduire à :

  • 💥 Code cassé
  • 🤯 Confusion
  • 🔍 Bugs cachés

🧪 Mauvais exemple : Violation de LSP

class Bird {
    void fly() {
        System.out.println("Flying...");
    }
}

class Ostrich extends Bird {
    @Override
    void fly() {
        throw new UnsupportedOperationException("Ostrich can't fly!");
    }
}

Cela rompt avec LSP. L'autruche est un oiseau, mais on ne peut pas le remplacer par Bird si la méthode fly() est censée toujours fonctionner.

✅ Conception adaptée aux FSL

Il faut plutôt restructurer la hiérarchie :

interface Bird {
    void makeSound();
}

interface FlyingBird extends Bird {
    void fly();
}

class Sparrow implements FlyingBird {
    public void fly() { ... }
    public void makeSound() { ... }
}

class Ostrich implements Bird {
    public void makeSound() { ... }
}

Maintenant, vous n'assignez fly() qu'aux oiseaux qui peuvent réellement voler.

🧱 Principaux enseignements à tirer de l'application des FSL

  • Les sous-classes doivent respecter le contrat de la classe de base
  • Éviter de modifier les comportements attendus
  • Utiliser des interfaces pour séparer les capacités
  • Rédiger des tests unitaires pour vérifier le comportement interchangeable

📘 Recap

  • LSP = Les sous-classes devraient être utilisables partout où leur classe mère est attendue
  • La violation de la PSL conduit à un code imprévisible et fragile
  • Séparer les fonctionnalités à l'aide d'interfaces si nécessaire
