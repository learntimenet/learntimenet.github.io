🔍 Que sont les événements JavaScript ?

Les événements JavaScript sont des actions ou des événements qui se produisent dans le système pour lequel vous programmez. Ils peuvent être déclenchés par l'utilisateur (comme cliquer sur un bouton ou appuyer sur une touche) ou par le navigateur (comme le chargement d'une page ou d'une image). JavaScript peut répondre à ces événements par un code spécifique.

Exemple d'événement :

<button onclick="alert('Button clicked!')">Click Me</button>

Dans cet exemple, lorsque le bouton est cliqué, il déclenche l'événement onclick, qui affiche une alerte.

📅 Types d'événements

Il existe de nombreux types d'événements en JavaScript, que l'on peut classer en plusieurs grandes catégories :

1️⃣ Événements de la souris

  • click - Déclenché lorsqu'un bouton de la souris est cliqué
  • dblclick - Déclenché lorsqu'un bouton de la souris est doublement cliqué
  • mouseover - Déclenché lorsque le pointeur de la souris entre dans un élément
  • mouseout - Déclenché lorsque le pointeur de la souris quitte un élément
  • mousemove - Déclenché lorsque la souris passe au-dessus d'un élément

Exemple :

<button onmouseover="console.log('Mouse entered button')">Hover over me</button>

2️⃣ Événements clavier

  • keydown - Déclenché lorsqu'une touche est enfoncée
  • keyup - Déclenché lorsqu'une touche est relâchée
  • keypress - Déclenché lorsqu'une touche est pressée (obsolète, utiliser keydown ou keyup)

Exemple :

<input type="text" onkeydown="console.log('Key pressed')">

3️⃣ Événements relatifs aux fenêtres/documents

  • load - Déclenché lorsque le chargement de la page est terminé
  • resize - Déclenché lorsque la fenêtre est redimensionnée
  • scroll - Déclenché lorsque l'utilisateur fait défiler la page
  • focus - Déclenché lorsqu'un élément est mis au premier plan
  • blur - Déclenché lorsqu'un élément perd le focus

Exemple :

<script>
  window.onload = function() {
    console.log('Page loaded!');
  };
</script>

🛠️ Écoute d'événements

Un Event Listener est un moyen plus souple de gérer les événements. Vous pouvez attacher un écouteur d'événements à un élément, ce qui vous permet de répondre aux événements sans code en ligne dans le HTML.

Syntaxe :

element.addEventListener(event, function, useCapture);
  • événement - Le type d'événement (par exemple, clic, survol de la souris)
  • fonction - La fonction à exécuter lorsque l'événement se produit
  • useCapture - Paramètre optionnel permettant de contrôler la propagation des événements (généralement fixé à false)

Exemple :

<button id="myButton">Click Me</button>

<script>
  let button = document.getElementById("myButton");
  
  button.addEventListener("click", function() {
    alert("Button clicked using addEventListener!");
  });
</script>

Dans cet exemple, lorsque le bouton est cliqué, l'écouteur d'événement déclenche l'alerte.

🧑‍💻 Suppression de l'écoute d'événements

Vous pouvez également supprimer les auditeurs d'événements avec la méthode removeEventListener.

Exemple :

<button id="myButton">Click Me</button>

<script>
  let button = document.getElementById("myButton");

  function handleClick() {
    alert("Button clicked!");
  }

  button.addEventListener("click", handleClick);

  // To remove the event listener:
  // button.removeEventListener("click", handleClick);
</script>

💡 Propagation des événements

Les événements JavaScript se propagent dans le DOM en deux phases : la capture et le bouillonnement.

  • Capture : L'événement part de la racine du document et descend jusqu'à l'élément cible.
  • Bubbling : L'événement commence à l'élément cible et remonte jusqu'à la racine.

Vous pouvez contrôler ce comportement avec le paramètre useCapture dans addEventListener ou en utilisant stopPropagation() et preventDefault().

🧠 Recap

  • Les événements JavaScript sont des actions qui se produisent à la suite d'une interaction avec l'utilisateur ou d'un événement système.
  • Vous pouvez gérer les événements en utilisant des attributs d'événements en ligne (par exemple, onclick) ou en utilisant des écoutants d'événements.
  • Les types d'événements les plus courants sont les événements liés à la souris, les événements liés au clavier et les événements liés à la fenêtre ou au document.
  • La propagation des événements peut être contrôlée par les phases de capture et de bouillonnement.
