🔍 Qu'est-ce que const ?

Le mot-clé const est utilisé pour déclarer des constantes - des variables dont la valeur ne peut pas être réaffectée après avoir été définie.

const name = "Alice";
name = "Bob"; // ❌ Error: Assignment to constant variable

Une fois déclarée et initialisée, une variable const ne doit pas être réassignée.

📦 Champ d'application du bloc

Comme let, const est scopé par bloc, ce qui signifie qu'il n'existe que dans le bloc {} où il est déclaré.

{
  const age = 25;
  console.log(age); // ✅ 25
}
console.log(age); // ❌ ReferenceError: age is not defined

Cela permet d'éviter les conflits et les bogues dans les grands programmes.

❗ Doit être initialisé

Vous devez assigner une valeur lorsque vous déclarez une const. Sinon, vous obtiendrez une erreur.

const x; // ❌ SyntaxError: Missing initializer

Utilisation correcte :

const x = 10;

🧱 const avec les objets et les tableaux

Bien que const empêche la réaffectation de la variable référence, il ne rend pas la valeur immuable.

const person = {
  name: "Alex"
};

person.name = "Jordan"; // ✅ This is allowed
person = {}; // ❌ Error: Assignment to constant variable

Vous pouvez changer le contenu des objets ou des tableaux déclarés avec const, mais vous ne pouvez pas réassigner la variable elle-même.

✅ Quand utiliser const ?

Utilisez const quand :

  • La valeur ne doit jamais être réaffectée
  • Vous travaillez avec des valeurs fixes ou des données de configuration
  • Vous souhaitez protéger la référence d'une réaffectation accidentelle

🧠 const vs let vs var

(en anglais)
Mot clé Réassignable Block ScopedCas d'utilisation
const ❌ Non ✅ Oui Valeurs constantes, plus sûres
let ✅ Oui ✅ Oui Modifier les valeurs
var ✅ Oui ❌ Non Code héritage uniquement

📘 Recap

  • const déclare des constantes - aucune réaffectation n'est autorisée
  • C'est scopé en bloc, comme let
  • Doit être initialisé lors de la déclaration
  • Un choix sûr et clair pour des valeurs qui ne devraient pas changer
