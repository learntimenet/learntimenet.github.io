🔍 Qu'est-ce que let ?

Le mot-clé let est utilisé pour déclarer des variables en JavaScript. Il a été introduit dans ES6 (ECMAScript 2015) et est maintenant la façon standard de définir des variables qui peuvent changer plus tard.

let name = "Alice";
name = "Bob"; // This is allowed

Contrairement à const, les variables déclarées avec let peuvent être réassignées.

📦 Champ d'application du bloc

L'un des plus grands avantages de let par rapport à var est la portée au niveau du bloc. Cela signifie qu'une variable déclarée à l'intérieur de {} n'est disponible qu'à l'intérieur de ce bloc.

{
  let age = 30;
  console.log(age); // ✅ 30
}
console.log(age); // ❌ Error: age is not defined

Avec var, cette même variable serait accessible en dehors du bloc, ce qui peut entraîner des bogues.

🔁 Boucles internes

L'utilisation de let à l'intérieur des boucles crée une nouvelle variable à chaque itération, ce qui est très utile :

for (let i = 0; i < 3; i++) {
  console.log(i);
}
// i is not accessible here

L'utilisation de var rendrait i accessible même après la fin de la boucle.

⚠️ Comportement de levage

Les variables déclarées avec let sont hoisées, mais ne sont pas initialisées. Y accéder avant la déclaration provoque une ReferenceError.

console.log(x); // ❌ ReferenceError
let x = 5;

C'est ce qu'on appelle la zone morte temporelle.

✅ Quand utiliser let ?

Utilisez let quand :

  • Vous vous attendez à ce que la valeur change plus tard
  • Vous voulez un cadrage plus sûr à l'intérieur des blocs ou des boucles
  • Vous voulez un comportement plus moderne et plus prévisible que var

(en anglais)

📘 Recap

  • let déclare des variables qui peuvent changer
  • Il s'agit d'un bloc, contrairement à var.
  • Un code moderne plus sûr et plus prévisible
  • Éviter d'y accéder avant la déclaration (zone morte temporelle)
