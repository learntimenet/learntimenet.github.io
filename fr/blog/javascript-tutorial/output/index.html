  • Quand et où utiliser chaque méthode
  • Exemples réels avec code
  • Les erreurs courantes des débutants à éviter

🖥 Sortie console

La méthode console.log() est l'un des moyens les plus couramment utilisés pour afficher la sortie, en particulier pour le débogage.

console.log("Hello, console!");

Où cela s'affiche: Developer Console dans votre navigateur (généralement ouvert avec F12 ou un clic droit → Inspecter → Console)

📦 Boîtes d'alerte

alert() affiche une boîte de dialogue popup avec un message. Elle est utile pour une interaction très basique ou pour des tests.

alert("Hello, user!");

Note: Cette méthode interrompt l'expérience de l'utilisateur, c'est pourquoi elle n'est pas utilisée dans le code de production.

📄 Écrire sur la page web

Vous pouvez utiliser document.write() pour produire du contenu directement dans le code HTML d'une page.

document.write("This is written to the page.");

**Attention : si elle est utilisée après le chargement de la page, elle peut écraser l'ensemble du document.

✏️ Mise à jour des éléments HTML

La façon la plus pratique et la plus moderne d'afficher les résultats est de mettre à jour un élément de la page web à l'aide de innerHTML.

<p id="demo"></p>

<script>
  document.getElementById("demo").innerHTML = "Hello from JavaScript!";
</script>

Cette méthode est non intrusive, c'est-à-dire qu'elle ne s'affiche pas et ne remplace pas la page entière - ce qui est idéal pour le travail sur l'interface utilisateur.

🛠 Autres méthodes de sortie

  • confirm() - affiche une boîte de dialogue avec OK/Annuler
  • prompt() - demande une entrée à l'utilisateur
  • console.error() - affiche un message d'erreur
  • console.table() - affiche les données sous forme de tableau (idéal pour les tableaux et les objets)

⚠️ Erreurs courantes

  • Utiliser trop souvent alert() - cela devient vite ennuyeux
  • Oublier de cibler le bon élément avec getElementById()
  • Utilisation de document.write() après le chargement complet de la page (cela efface le contenu)

📘 Recap

  • Utiliser console.log() pour le débogage
  • Utilisez alert() avec parcimonie, principalement pour l'apprentissage.
  • Préférer innerHTML pour afficher la sortie dans l'interface utilisateur
  • Éviter document.write() dans les projets réels
