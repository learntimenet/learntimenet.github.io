Introduction

Git is the most widely used version control system, enabling developers to track changes, collaborate efficiently, and manage codebases effectively. Whether you're a beginner or an experienced developer, having a quick reference to Git commands can save time and streamline your workflow. This Git Cheat Sheet provides a comprehensive list of essential Git commands, tips, and best practices.

Git Basics

This section covers the foundational commands you need to get started with Git. These commands allow you to initialize a repository, track changes, and commit your work. Whether you're setting up a new project or cloning an existing one, these commands are essential for every developer.

cheatsheet

  • Create a new Git repository in your project directory
    git init
  • Clone an existing repository from a remote URL
    git clone <repository_url>
  • View the status of your working directory
    git status
  • Stage changes for the next commit
    git add <file_name>
  • Stage all changes
    git add .
  • Commit staged changes with a message
    git commit -m "Your commit message"
  • Display the commit history
    git log

Branching and Merging

Branching and merging are core features of Git that allow developers to work on multiple features or fixes simultaneously without affecting the main codebase. Branches are lightweight and easy to create, making them ideal for isolating changes. Merging combines changes from different branches, ensuring a smooth integration of code.

cheatsheet

  • Create a new branch
    git branch <branch_name>
  • Switch to an existing branch
    git checkout <branch_name>
  • Create and switch to a new branch in one command
    git checkout -b <branch_name>
  • View all branches in the repository
    git branch
  • Merge a branch into the current branch
    git merge <branch_name>
  • Delete a branch
    git branch -d <branch_name>

Remote Repositories

Remote repositories allow you to collaborate with others by pushing and pulling changes to and from a shared repository. Git makes it easy to manage remote repositories, enabling seamless collaboration across teams and projects.

cheatsheet

  • Add a remote repository
    git remote add <remote_name> <repository_url>
  • List all remote repositories
    git remote -v
  • Fetch changes from a remote repository
    git fetch <remote_name>
  • Pull changes from a remote branch
    git pull <remote_name> <branch_name>
  • Push changes to a remote branch
    git push <remote_name> <branch_name>
  • Remove a remote repository
    git remote remove <remote_name>

Undoing Changes

Mistakes happen, and Git provides several ways to undo changes. Whether you need to unstage a file, revert a commit, or reset your repository to a previous state, Git has you covered.

cheatsheet

  • Unstage a file while keeping changes
    git reset <file_name>
  • Revert changes in a file to the last commit
    git checkout -- <file_name>
  • Modify the last commit
    git commit --amend
  • Revert a specific commit
    git revert <commit_hash>
  • Reset the repository to a previous commit
    git reset --hard <commit_hash>

Stashing

Stashing allows you to temporarily save changes that are not ready to be committed. This is useful when you need to switch branches or work on something else without losing your current progress.

cheatsheet

  • Stash changes in the working directory
    git stash
  • View all stashes
    git stash list
  • Apply the most recent stash
    git stash apply
  • Apply a specific stash
    git stash apply stash@{<stash_index>}
  • Delete the most recent stash
    git stash drop
  • Delete a specific stash
    git stash drop stash@{<stash_index>}

Logs and History

Git logs provide a detailed history of commits, making it easy to track changes, identify issues, and understand the evolution of your codebase. You can view logs in various formats to suit your needs.

cheatsheet

  • Display the commit history
    git log
  • Display a simplified commit history
    git log --oneline
  • View changes made in a specific commit
    git show <commit_hash>
  • View changes between two commits
    git diff <commit_hash_1> <commit_hash_2>

Tagging

Tags are used to mark specific points in your repository's history, such as releases or milestones. They provide a way to easily reference important commits.

cheatsheet

  • Create a lightweight tag
    git tag <tag_name>
  • Create an annotated tag
    git tag -a <tag_name> -m "Tag message"
  • View all tags
    git tag
  • Push tags to a remote repository
    git push <remote_name> --tags
  • Delete a tag
    git tag -d <tag_name>

Advanced Git Commands

Advanced Git commands provide powerful tools for managing complex workflows, debugging, and optimizing your repository. These commands are essential for experienced developers working on large projects.

cheatsheet

  • Rebase the current branch onto another branch
    git rebase <branch_name>
  • Start an interactive rebase
    git rebase -i <commit_hash>
  • Apply a specific commit to the current branch
    git cherry-pick <commit_hash>
  • Push tags to a remote repository
    git push <remote_name> --tags
  • Delete a tag
    git tag -d <tag_name>

Tips and Best Practices

Following best practices ensures a smooth and efficient Git workflow. These tips will help you avoid common pitfalls and make the most of Git's features.

  • Write Meaningful Commit Messages: Use clear and concise commit messages to describe changes.
  • Use Branches for Features: Create separate branches for new features or bug fixes.
  • Pull Before Pushing: Always pull the latest changes before pushing to avoid conflicts.
  • Review Changes Before Committing: Use git diff to review changes before committing.
  • Use .gitignore: Exclude unnecessary files from version control using a .gitignore file.
