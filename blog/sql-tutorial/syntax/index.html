üìê Basic SQL Statement Structure

SQL follows a straightforward syntax that's easy to read and write.

üß± Common SQL Keywords

SQL keywords are reserved words that tell the database what action to perform.

Here are some commonly used SQL keywords:

Keyword Description
SELECT Retrieves data from a table
FROM Specifies the table
WHERE Filters rows based on a condition
INSERT INTO Adds new rows to a table
UPDATE Modifies existing data
DELETE Removes rows from a table
CREATE TABLE Creates a new table
ALTER TABLE Modifies a table‚Äôs structure
DROP TABLE Deletes a table completely

üß† Tip: Even though SQL keywords are not case-sensitive, writing them in uppercase improves readability.

üî§ Case Sensitivity in SQL

  • Keywords like SELECT, FROM, and WHERE are not case-sensitive.
  • Table and column names may be case-sensitive, depending on the database system (e.g., MySQL on Linux = case-sensitive).
SELECT Name FROM Employees;
-- Same as
select name from employees;

‚úÖ Best Practice: Use uppercase for SQL keywords and lowercase or camelCase for table/column names.

üßæ Do You Need a Semicolon?

Yes! The semicolon (;) is used to end an SQL statement, especially when writing multiple queries in a script.

Some SQL environments may not require it for single queries, but it‚Äôs a good habit to always include it.
