Why Structure Matters

When your folder structure grows organically without guidelines, you risk:

  • Performance issues ‚Äî importing heavy backend logic into frontend bundles.

A good structure:

The Three Types of Code in Next.js

When you think about where code runs in a Next.js project, you can group it into three categories:

1. Server-Only Code

Runs only in Node.js (never shipped to the browser).
Use it for:

  • Database queries
  • Authentication logic
  • File system access
  • Email sending

‚úÖ Safe to import server-only libraries (e.g., Prisma, AWS SDK).

2. Client-Only Code

Runs only in the browser.
Use it for:

  • Interactivity (useState, useEffect, event handlers)
  • DOM APIs (window, document)
  • Local storage

‚úÖ Must include "use client" at the top in the App Router.

3. Shared (Isomorphic) Code

Runs in both environments.
Use it for:

  • TypeScript types
  • Date formatting
  • Validation schemas
  • Pure UI components (no server APIs, no browser APIs)

Pro Tips for Safe Separation

Mark server-only files

Mark client components
