üîß Basic Syntax

for (initialization; condition; update) {
    // code to run each time
}

Each part of the loop plays a role:

  • Initialization: Set a starting value (runs once)
  • Condition: Loop continues while true
  • Update: Changes the loop variable (runs after each loop)

‚úÖ Example: Count to 5

for (int i = 1; i <= 5; i++) {
    System.out.println("i = " + i);
}

üì§ Output:

i = 1
i = 2
i = 3
i = 4
i = 5

‚ûø Nested For Loops

Loops inside loops ‚Äî commonly used for grids, tables, or patterns.

for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 2; j++) {
        System.out.println("i=" + i + ", j=" + j);
    }
}

üõë Using break and continue

break ‚Äì exit the loop early:

for (int i = 1; i <= 5; i++) {
    if (i == 3) break;
    System.out.println(i);
}

continue ‚Äì skip the current iteration:

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;
    System.out.println(i);
}

üîÑ Enhanced For Loop (For-Each)

Best for arrays and collections:

int[] numbers = {10, 20, 30};

for (int num : numbers) {
    System.out.println(num);
}

No index tracking needed ‚Äî just loop through each element.

‚ö†Ô∏è Common Pitfalls

  • Off-by-one errors (i <= 5 vs i < 5)
  • Infinite loops if update is missing or wrong
  • Modifying a collection while looping over it (with for-each)

üìò Recap

  • for loops are best when you know how many times to iterate
  • Built-in structure: init ‚Üí condition ‚Üí update
  • for-each is clean for arrays and lists
  • Use break and continue to control flow
