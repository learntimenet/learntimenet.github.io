üìö What is a Package?

A package in Java is like a folder that groups related classes and interfaces together.

Why use packages?

‚úÖ Organize code
‚úÖ Avoid class name conflicts
‚úÖ Control access and visibility
‚úÖ Reuse code across projects

üèóÔ∏è Creating a Package

Let‚Äôs create a package called shapes:

üìÅ File: shapes/Circle.java

package shapes;

public class Circle {
    public double area(double radius) {
        return Math.PI * radius * radius;
    }
}
  • The first line declares the package
  • File must be saved in a folder named shapes

üîÑ Importing a Package

To use Circle in another file:

üìÅ File: Main.java

import shapes.Circle;

public class Main {
    public static void main(String[] args) {
        Circle c = new Circle();
        System.out.println("Area: " + c.area(5));
    }
}

Use import shapes.*; to import all classes in a package.

üß± Static Imports

Use static import to call static members without class name:

import static java.lang.Math.*;

public class Example {
    public static void main(String[] args) {
        System.out.println(sqrt(16));  // instead of Math.sqrt()
    }
}

üß† Best Practices

  • Use lowercase for package names: com.myapp.utils
  • Avoid name clashes (use domain-style: com.example.app)
  • Group related classes in the same package
  • Keep public APIs in clearly defined packages

üìò Recap

  • Packages help structure Java code into logical groups
  • Use package to define, and import to use
  • Java provides many helpful built-in packages
  • You can create custom packages to organize your codebase
