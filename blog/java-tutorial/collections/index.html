üß† What is the Collections Framework?

The Collections Framework is a set of classes and interfaces in java.util that helps you work with groups of objects like lists of names, sets of tags, or maps of users and passwords.

üîë Core Interfaces

Interface Description Example Class
List Ordered collection, allows duplicates ArrayList, LinkedList
Set Unordered, no duplicates HashSet, TreeSet
Map Key-value pairs HashMap, TreeMap
Queue First-in, first-out collection LinkedList, PriorityQueue

üìã Working with Lists

List<String> fruits = new ArrayList<>();
fruits.add("Apple");
fruits.add("Banana");
System.out.println(fruits.get(0));  // Apple

Use ArrayList for fast access, LinkedList for fast inserts/removals.

üß© Sets ‚Äì No Duplicates

Set<String> colors = new HashSet<>();
colors.add("Red");
colors.add("Green");
colors.add("Red");  // ignored
System.out.println(colors);  // [Red, Green]

Use TreeSet if you want automatic sorting.

üîë Maps ‚Äì Key-Value Pairs

Map<String, Integer> scores = new HashMap<>();
scores.put("Alice", 90);
scores.put("Bob", 85);
System.out.println(scores.get("Alice"));  // 90

‚úÖ HashMap for general use
‚úÖ LinkedHashMap to preserve insertion order
‚úÖ TreeMap for sorted keys

üì¶ Queues ‚Äì FIFO

Queue<String> tasks = new LinkedList<>();
tasks.offer("Task1");
tasks.offer("Task2");
System.out.println(tasks.poll());  // Task1

Use PriorityQueue to automatically sort items by priority.

üîÅ Iterating Over Collections

for (String fruit : fruits) {
    System.out.println(fruit);
}

Or using an iterator:

while (it.hasNext()) {
    System.out.println(it.next());
}
