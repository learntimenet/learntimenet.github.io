üß± Inside the Section

You can place JavaScript inside the <head> tag using a <script> tag:

<head>
  <script>
    console.log("This runs before the body loads");
  </script>
</head>

Note: If the script tries to interact with elements in the body before they are loaded, it may not work correctly unless wrapped in a function like window.onload.

üìÑ Inside the Section

Placing JavaScript at the end of the <body> ensures all HTML is loaded before the script runs:

<body>
  <h1>Hello World</h1>

  <script>
    console.log("This runs after the page is loaded");
  </script>
</body>

üìÅ External JavaScript Files

You can keep your JavaScript in a separate file (e.g., script.js) and include it with the src attribute:

<script src="script.js"></script>

This keeps your HTML clean and allows you to reuse scripts across multiple pages.

Example:

<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
    <script src="script.js"></script>
  </head>
  <body>
    <h1>Welcome!</h1>
  </body>
</html>

‚è≥ Using defer and async

defer

  • Loads the script in the background while parsing HTML
  • Executes the script after the HTML is fully parsed
  • Recommended for most external scripts
<script src="script.js" defer></script>

async

  • Loads the script in the background
  • Executes the script as soon as it‚Äôs ready (may interrupt HTML parsing)
  • Use only when the script is independent
<script src="script.js" async></script>

üõ† Best Practices

  • Place <script> tags at the end of <body> or use defer in <head>
  • Use external scripts for maintainability
  • Avoid document.write() in modern pages
  • Only use async if the script doesn't depend on or interfere with other scripts or the page content
