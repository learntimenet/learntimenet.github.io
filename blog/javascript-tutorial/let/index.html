üîç What Is let?

The let keyword is used to declare variables in JavaScript. It was introduced in ES6 (ECMAScript 2015) and is now the standard way to define variables that may change later.

let name = "Alice";
name = "Bob"; // This is allowed

Unlike const, variables declared with let can be reassigned.

üì¶ Block Scope

One of the biggest advantages of let over var is block-level scope. This means that a variable declared inside {} is only available within that block.

{
  let age = 30;
  console.log(age); // ‚úÖ 30
}
console.log(age); // ‚ùå Error: age is not defined

With var, that same variable would be accessible outside the block, which can lead to bugs.

üîÅ Inside Loops

Using let inside loops creates a new variable for each iteration, which is very useful:

for (let i = 0; i < 3; i++) {
  console.log(i);
}
// i is not accessible here

Using var would make i accessible even after the loop ends.

‚ö†Ô∏è Hoisting Behavior

Variables declared with let are hoisted, but not initialized. Accessing them before declaration causes a ReferenceError.

console.log(x); // ‚ùå ReferenceError
let x = 5;

This is known as the temporal dead zone.

‚úÖ When to Use let

Use let when:

  • You expect the value to change later
  • You want safer scoping inside blocks or loops
  • You want more modern and predictable behavior than var
