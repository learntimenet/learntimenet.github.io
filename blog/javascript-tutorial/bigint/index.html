🔍 What Is BigInt?

BigInt is a built-in object in JavaScript that allows you to work with arbitrary-precision integers—numbers that are too large or too small to be represented by the Number type. While JavaScript's Number type can only handle values up to approximately 2^53 - 1 (the largest safe integer), BigInt allows you to represent and perform operations on much larger numbers without losing precision.

Example:

let bigNumber = 1234567890123456789012345678901234567890n;
console.log(bigNumber);  // Output: 1234567890123456789012345678901234567890n

Note the n at the end of the number, indicating that it is a BigInt.

🧑‍🍳 Creating BigInt

You can create a BigInt in JavaScript by appending the letter n to an integer or using the BigInt() constructor.

1️⃣ Using n to Create BigInt

let bigIntValue = 9876543219876543210n;
console.log(bigIntValue);  // Output: 9876543219876543210n

2️⃣ Using BigInt() Constructor

let bigIntFromConstructor = BigInt("9876543219876543210");
console.log(bigIntFromConstructor);  // Output: 9876543219876543210n

Both methods create a BigInt value. The difference is that BigInt() can also be used to convert strings or other numeric values into a BigInt.

🧮 BigInt Operations

You can perform mathematical operations on BigInt values just like regular numbers, such as addition, subtraction, multiplication, and division.

1️⃣ Addition and Subtraction

let bigNum1 = 12345678901234567890n;
let bigNum2 = 98765432109876543210n;

let sum = bigNum1 + bigNum2;
let diff = bigNum2 - bigNum1;

console.log(sum);   // Output: 111111111011111111100n
console.log(diff);  // Output: 86419753208641975320n

2️⃣ Multiplication

let bigNum1 = 12345678901234567890n;
let bigNum2 = 98765432109876543210n;

let product = bigNum1 * bigNum2;
console.log(product);  // Output: 1219326311370217952274851956012610475690n

3️⃣ Division

BigInt division truncates the result toward zero (removes the fractional part).

let bigNum1 = 12345678901234567890n;
let bigNum2 = 98765432109876543210n;

let quotient = bigNum2 / bigNum1;
console.log(quotient);  // Output: 8n

4️⃣ Modulo (Remainder)

let bigNum1 = 12345678901234567890n;
let bigNum2 = 98765432109876543210n;

let remainder = bigNum2 % bigNum1;
console.log(remainder);  // Output: 86419753208641975320n

🛑 Limitations of BigInt

While BigInt is useful for handling large integers, it has a few limitations:

1️⃣ Mixed Operations with Numbers

You cannot mix BigInt and Number types in operations directly. For example, you cannot add a BigInt and a Number together:

let bigIntValue = 12345678901234567890n;
let numberValue = 10;

let result = bigIntValue + numberValue;  // Error: Cannot mix BigInt and other types

To solve this, you need to convert the Number to BigInt first:

let result = bigIntValue + BigInt(numberValue);  // Correct
console.log(result);  // Output: 12345678901234567890n

2️⃣ No Decimal Support

BigInt can only represent whole numbers (integers) and does not support floating-point values or decimals.

let floatValue = 10.5;
let bigIntValue = BigInt(floatValue);  // Error: Cannot convert a non-integer value to BigInt

To handle decimals, you must use regular Numbers or other methods of handling floating-point arithmetic.

💡 When to Use BigInt?

BigInt is ideal for situations that involve very large integers or situations where you need arbitrary precision. Examples include:

  • Cryptographic applications
  • Financial applications that require high-precision calculations
  • Handling large scientific data

However, in most everyday scenarios, the built-in Number type in JavaScript is sufficient.

🧠 Recap

  • BigInt allows you to work with integers beyond the limits of the Number type.
  • You can create BigInt using the n suffix or the BigInt() constructor.
  • Mathematical operations on BigInt work the same as regular numbers, but mixing BigInt with Number types directly is not allowed.
  • BigInt does not support floating-point numbers or decimals.
