üîç What Is the Facade Pattern?

The Facade Pattern provides a unified, simplified interface to a set of interfaces in a subsystem. It hides the complexities of the system and provides an easier way to interact with it.

Like a hotel concierge ‚Äî you don‚Äôt need to know all departments; the concierge handles it for you.

üß© When to Use It

Use the Facade Pattern when:

  • A system is complex or has multiple interdependent components
  • You want to provide a simple interface for common tasks
  • You want to decouple client code from subsystem code

üß± Step 2: Create the Facade

public class HomeTheaterFacade {
    private DVDPlayer dvd;
    private Projector projector;
    private SoundSystem sound;

    public HomeTheaterFacade(DVDPlayer dvd, Projector projector, SoundSystem sound) {
        this.dvd = dvd;
        this.projector = projector;
        this.sound = sound;
    }

    public void watchMovie(String movie) {
        System.out.println("Get ready to watch a movie...");
        projector.on();
        sound.on();
        sound.setVolume(5);
        dvd.on();
        dvd.play(movie);
    }

    public void endMovie() {
        System.out.println("Shutting down movie theater...");
        dvd.off();
        sound.off();
        projector.off();
    }
}

  • ‚ùå Over-simplifying to the point of removing needed functionality

‚úÖ Best Practices

  • Use it to organize and decouple code, not hide poor structure
