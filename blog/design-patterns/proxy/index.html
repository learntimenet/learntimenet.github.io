üîç What Is the Proxy Pattern?

The Proxy Pattern provides a substitute or placeholder for another object to control access, add functionality, or delay expensive operations until needed.

Think of a security guard (proxy) checking credentials before letting you into a building (real object).

üß© When to Use It

Use the Proxy Pattern when you need:

  • üîê Access control (e.g., authentication, authorization)
  • üí§ Lazy initialization (e.g., only load when accessed)
  • üßæ Logging or caching without modifying the original class
  • üõ∞Ô∏è Remote proxying for distributed objects

    public void grantInternetAccess() {

            realAccess.grantInternetAccess();
        } else {
            System.out.println("No Internet access granted to " + employeeName);
        }
    }

    private int getRole(String empName) {
        // Simulate role check: higher number means higher privileges
        return empName.length(); // just for demo purposes
    }
}
