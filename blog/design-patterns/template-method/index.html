üîç What Is the Template Method Pattern?

The Template Method Pattern defines the skeleton of an algorithm in a method, deferring some steps to subclasses.
Subclasses can redefine certain steps of an algorithm without changing its structure.

Think about making tea or coffee: the steps are similar (boil water, brew, pour, add condiments), but some steps vary slightly.

üß© When to Use It

You should use the Template Method Pattern when:

  • You have multiple classes with similar algorithms but some differences in the steps
  • You want to avoid code duplication
  • You need a consistent algorithm structure, but flexible individual steps

üß± Java Implementation

üë©‚Äçüè´ Step 1: Define the Abstract Class

public abstract class Beverage {
    
    // Template method
    public final void prepareRecipe() {
        boilWater();
        brew();
        pourInCup();
        addCondiments();
    }

    abstract void brew();
    abstract void addCondiments();

    public void boilWater() {
        System.out.println("Boiling water...");
    }

    public void pourInCup() {
        System.out.println("Pouring into cup...");
    }
}

‚òï Step 2: Concrete Implementations

public class Tea extends Beverage {

    void brew() {
        System.out.println("Steeping the tea...");
    }

    void addCondiments() {
        System.out.println("Adding lemon...");
    }
}

public class Coffee extends Beverage {

    void brew() {
        System.out.println("Dripping coffee through filter...");
    }

    void addCondiments() {
        System.out.println("Adding sugar and milk...");
    }
}

‚úÖ Usage Example

public class TemplateMethodPatternDemo {
    public static void main(String[] args) {
        Beverage tea = new Tea();
        tea.prepareRecipe();

        System.out.println();

        Beverage coffee = new Coffee();
        coffee.prepareRecipe();
    }
}

üåê Real-World Analogy

  • Making tea and coffee: Same sequence, but "brew" and "add condiments" steps differ.
  • Cooking recipes: Bake a cake, but ingredients and flavorings differ.
  • Document generation: Common steps with slight variations for different document types.

‚ö†Ô∏è Pitfalls to Avoid

  • ‚ùå Making the base class too rigid (subclasses must have flexibility)
  • ‚ùå Using Template Method where simple inheritance or interface would suffice
  • ‚ùå Letting subclasses violate the sequence ‚Äî the skeleton should be consistent

‚úÖ Best Practices

  • Use final on the template method to prevent overriding
  • Keep abstract methods clear and minimal
  • Combine with hooks (optional steps) if needed for flexibility

üìò Recap
