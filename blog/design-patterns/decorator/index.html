🔍 What Is the Decorator Pattern?

The Decorator Pattern lets you attach new behaviors to objects at runtime by placing them inside wrapper objects, or "decorators", that implement the same interface.

It’s a flexible alternative to subclassing for extending functionality.

🧩 When to Use It

Use the Decorator Pattern when:

  • You need to add features to objects without changing their class
  • You want to avoid creating too many subclasses
  • You want to layer behaviors flexibly and dynamically

🧱 Java Implementation

🧩 Step 1: Define the Component Interface

public interface Coffee {
    String getDescription();
    double getCost();
}

☕ Step 2: Create a Concrete Component

public class BasicCoffee implements Coffee {
    public String getDescription() {
        return "Basic Coffee";
    }

    public double getCost() {
        return 2.0;
    }
}

🧥 Step 3: Abstract Decorator

public abstract class CoffeeDecorator implements Coffee {
    protected Coffee decoratedCoffee;

    public CoffeeDecorator(Coffee coffee) {
        this.decoratedCoffee = coffee;
    }

    public String getDescription() {
        return decoratedCoffee.getDescription();
    }

    public double getCost() {
        return decoratedCoffee.getCost();
    }
}

🍫 Step 4: Concrete Decorators

public class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }

    public String getDescription() {
        return super.getDescription() + ", Milk";
    }

    public double getCost() {
        return super.getCost() + 0.5;
    }
}


public class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) {
        super(coffee);
    }

    public String getDescription() {
        return super.getDescription() + ", Sugar";
    }

    public double getCost() {
        return super.getCost() + 0.2;
    }
}

✅ Usage

Coffee coffee = new BasicCoffee();
coffee = new MilkDecorator(coffee);
coffee = new SugarDecorator(coffee);

System.out.println(coffee.getDescription()); // Basic Coffee, Milk, Sugar
System.out.println("Total: __HTML__quot; + coffee.getCost()); // Total: $2.7

☕ Real-World Analogy

Imagine ordering a coffee: you start with a basic cup and then decorate it with milk, sugar, whipped cream, etc. Each add-on wraps the coffee and adds its own flavor (and price)!

⚠️ Pitfalls to Avoid

  • ❌ Making decorators too complex — keep each small and focused
  • ❌ Forgetting to delegate methods to the wrapped component
  • ❌ Over-using when subclassing would be simpler

✅ Best Practices

  • Keep decorators lightweight and single-purpose
  • Stick to the Open/Closed Principle: open for extension, closed for modification
  • Chain decorators in the order they should apply

📘 Recap

  • Decorator Pattern allows dynamic extension of object behavior
  • Useful alternative to deep inheritance trees
  • Great for UI components, input/output streams, and feature toggles
  • Easy to mix and match new functionality
