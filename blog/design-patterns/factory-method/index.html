üîç What Is the Factory Method Pattern?

The Factory Method Pattern defines an interface for creating an object, but allows subclasses to decide which class to instantiate.

It promotes loose coupling by avoiding direct object creation.

üéØ When Should You Use Factory Method?

Use Factory Method when:

  • Your code needs to decide which class to instantiate at runtime
  • You want to hide object creation logic
  • You‚Äôre dealing with families of related objects
  • You need to adhere to the Open/Closed Principle

            case "email": return new EmailNotification();
            default: return null;
        }
    }
