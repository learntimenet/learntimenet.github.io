üîç What Is the Bridge Pattern?

The Bridge Pattern is used to decouple an abstraction from its implementation so that the two can vary independently.

Imagine a TV remote: different remotes can control different TVs, but the remote (abstraction) and the TV (implementation) can evolve separately.

üß© When to Use It

Use the Bridge Pattern when:

  • You want to avoid a permanent binding between abstraction and implementation
  • You have multiple variations (e.g., types of devices and ways to control them)
  • You want to reduce complexity caused by many subclasses (like a huge class hierarchy)
  • You want to be able to extend both abstraction and implementation independently

üß± Java Implementation

üîå Step 1: Create the Implementation Interface

public interface Device {
    void turnOn();
    void turnOff();
    void setVolume(int level);
}

üì∫ Step 2: Concrete Implementations

public class TV implements Device {
    public void turnOn() {
        System.out.println("TV is turned ON");
    }

    public void turnOff() {
        System.out.println("TV is turned OFF");
    }

    public void setVolume(int level) {
        System.out.println("TV volume set to " + level);
    }
}


public class Radio implements Device {
    public void turnOn() {
        System.out.println("Radio is turned ON");
    }

    public void turnOff() {
        System.out.println("Radio is turned OFF");
    }

    public void setVolume(int level) {
        System.out.println("Radio volume set to " + level);
    }
}

üéÆ Step 3: Create the Abstraction

public abstract class RemoteControl {
    protected Device device;

    protected RemoteControl(Device device) {
        this.device = device;
    }

    public abstract void turnOn();
    public abstract void turnOff();
    public abstract void setVolume(int level);
}

üéõÔ∏è Step 4: Concrete Remotes

public class BasicRemote extends RemoteControl {

    public BasicRemote(Device device) {
        super(device);
    }

    public void turnOn() {
        device.turnOn();
    }

    public void turnOff() {
        device.turnOff();
    }

    public void setVolume(int level) {
        device.setVolume(level);
    }
}

üåê Real-World Analogy

  • Remote controls and devices: Same remote (abstraction) can control different devices (TV, Radio, Sound System).
  • Web browsers and operating systems: The browser (abstraction) can work across different OS platforms (implementations).

‚ö†Ô∏è Pitfalls to Avoid

  • ‚ùå Introducing unnecessary complexity if only one abstraction or implementation exists
  • ‚ùå Tight coupling if abstraction relies on specific implementation details
  • ‚ùå Overengineering when a simple inheritance would work

‚úÖ Best Practices

  • Prefer composition over inheritance
  • Allow abstraction and implementation to evolve independently
  • Clearly separate what the abstraction and implementation each manage

üìò Recap

  • Bridge Pattern decouples abstraction from implementation
  • Allows for flexible extension of both sides without affecting the other
  • Reduces the complexity of a huge class hierarchy
