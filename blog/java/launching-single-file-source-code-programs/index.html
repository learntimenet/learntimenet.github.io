Java has long been celebrated for its robustness and portability. With the introduction of JDK 11, Java further enhances developer productivity by allowing the execution of single-file source-code programs without the need for explicit compilation. This feature simplifies the development process, making it more accessible for beginners and efficient for seasoned developers.

üõ†Ô∏è What Is Single-File Source-Code Execution?

Traditionally, running a Java program involves two steps: compiling the .java file into bytecode using javac, and then executing the compiled .class file with the java comman. JDK 11 introduces a streamlined approach where you can execute a Java source file directly:

java HelloWorld.java

Under the hood, the java launcher automatically compiles the source code in-memory and then executes it, eliminating the need for manual compilation.

‚úçÔ∏è Writing and Running Your First Single-File Program

Let's create a simple Java program that prints "Hello, World!" to the console.

1. Create the Java Source File

Open your preferred text editor and write the following code:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Save this file as HelloWorld.java.

2. Execute the Program

Open your terminal, navigate to the directory containing HelloWorld.java, and run:

java HelloWorld.java

You should see the output:

Hello, World!

üß© Passing Command-Line Arguments

You can pass arguments to your program just like with traditionally compiled Java programs.

public class GreetUser {
    public static void main(String[] args) {
        System.out.println("Hello, " + args[0] + "!");
    }
}

Execute the program with an argument:

java GreetUser.java Alice

Output:

Hello, Alice!

üß± Multiple Classes in a Single Fie

You can define multiple classes within the same source file. Only the first public class with the main method will be executed.

public class MainApp {
    public static void main(String[] args) {
        Helper.greet();
    }
}

class Helper {
    static void greet() {
        System.out.println("Greetings from Helper!");
   }
}

Run the program:

java MainApp.ava

Output:

Greetings from Heler!

On Unix-like systems, you can create executable Java scripts using a shebang line.

  1. Create a file without the .java extension, for example, greet.

  2. Add the following cotent:

#!/usr/bin/env java --source 11

public class Greet {
    public static void main(String[] args) {
        System.out.println("Hello from a script!");
   }
}
  1. Make the script execuable:
chmod +x greet
  1. Run the sript:
./greet

Output:

Hello from a script!

Note: The file must not have a .java extension and should be execuable.

‚ö†Ô∏è Considerations and Limitations

  • Single Fie: This feature is designed for single-file programs. For multi-file projects, traditional compilation is recommended.

  • Performane: Since the source is compiled each time you run it, execution may be slower compared to pre-compiled classes.

  • Complex Projecs: For larger applications with multiple dependencies and modules, it's advisable to use build tools like Maven or Gradle.
